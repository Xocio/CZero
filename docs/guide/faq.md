# 常见问题

## 修改配置后需要重启吗？

不需要。守护进程监视 `config.json` 的变更并在下一分钟热重载调度；启用项、阈值等非调度字段由各清理组件运行时自读，即时生效。手动改动请用原子写入（先写临时文件再覆盖）。

## 我的设备不是 F2FS，能用吗？

可以。除 **F2FS GC** 外的所有功能都不依赖文件系统类型。非 F2FS 设备将 `gc.enabled` 设为 `false` 即可，其余功能照常工作。

## 不安装 CZeroX 可以吗？

可以。模块会按默认 `config.json` 正常运行。CZeroX 的作用是让配置与状态查看变得直观；不安装则需手动编辑 `config.json`。

## 支持哪些 Root 方案？

Magisk、KernelSU、APatch 均可。

## 重装模块会丢失我的设置吗？

安装时可选择继承旧配置，此时会沿用黑白名单与自定义路径列表；但 `config.json` 始终重置为新版默认值，需要的话重装后再调整。

## 配置写错了会怎样？

若整份 `config.json` 无法解析，守护进程会**保留上一份有效任务集**，不会中断服务。若只是某个任务的 `every` 非法，则**仅跳过该任务**的调度，其余照常。

## 日志在哪里？

开启 `general.log` 后，日志位于 `/data/adb/modules/CZero/log/<YYYY-MM-DD>.log`，只保留当天。每条日志带来源标签，便于区分是哪个组件产生的。

## 清理会误删我的数据吗？

清理仅针对缓存与残留目录，并受 `temporal_barrier_days`（时序屏障）保护 —— 只处理 N 天以前的文件。如需更保守，可调大该值或缩小清理范围。

## 遇到问题去哪里反馈？

请到 [GitHub Issues](https://github.com/Xocio/CZero/issues) 提交。
